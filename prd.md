# 日照金山概率小程序 PRD

## 1. 产品概述

本产品是一款面向游客的微信小程序，用于查询指定雪山/观测点未来日出时段出现“日照金山”的概率。产品通过整合 EC、GFS、ICON 等气象模型数据，结合固定点位信息、日出日落时间与后续概率评分模型，为用户提供明日日出概率、七日趋势与简单明确的出行建议。

V1 版本定位为“游客决策助手”，帮助用户快速判断某个标杆点位明天早上是否值得前往观测，而不是面向专业用户的气象分析工具。

## 2. 产品目标

- 让游客在进入小程序后，可以快速选择目标雪山/观测点。
- 让用户在点位详情页一眼看到明日日出概率、推荐标签、日出日落时间和七日趋势。
- 通过七日预报辅助用户做短期旅行安排，其中前 3 天偏预测，后 4 天偏趋势参考。
- 保留 EC、GFS、ICON 三种气象模型数据的展开入口，为后续解释概率结果提供依据。
- 在设置页提供数据来源说明、免责声明和反馈入口，降低用户误解和产品风险。

## 3. 目标用户

### 3.1 核心用户

普通游客、轻量摄影爱好者、计划前往雪山观景点的旅行用户。

### 3.2 用户特征

- 希望快速知道“明天早上值不值得去看日照金山”。
- 不具备专业气象知识，不希望阅读复杂气象图表。
- 更关注概率、推荐程度、观测时间和趋势变化。
- 可能在出行前一天或旅行途中临时查询。

### 3.3 非目标用户

V1 暂不面向专业气象人员、重度户外玩家、商业摄影团队或需要精细气象分析的用户。

## 4. V1 范围

V1 先支持 9 个标杆点位，不开放地图任意选点预测。

### 4.1 标杆点位

1. 梅里雪山
2. 贡嘎雪山
3. 玉龙雪山
4. 南迦巴瓦峰
5. 四姑娘山
6. 珠穆朗玛峰
7. 亚丁三神山
8. 冈仁波齐
9. 慕士塔格峰

### 4.2 V1 核心能力

- 点位列表浏览
- 点位详情查看
- 明日日出概率展示
- 概率推荐标签展示
- 日出时间与日落时间展示
- 七日预报趋势展示
- 点击七日预报某一天后，展开当天 EC、GFS、ICON 三种气象模型数据
- 设置页展示产品说明、数据来源、免责声明与反馈入口

## 5. 页面结构

V1 包含 3 个主要页面：

1. 点位列表页
2. 点位详情页
3. 设置页

## 6. 点位列表页

### 6.1 页面目标

让用户快速选择想要查看的雪山/观测点。

### 6.2 展示内容

点位列表页 V1 仅展示点位名称，保持入口简洁。

示例：

- 梅里雪山
- 贡嘎雪山
- 玉龙雪山
- 南迦巴瓦峰
- 四姑娘山
- 珠穆朗玛峰
- 亚丁三神山
- 冈仁波齐
- 慕士塔格峰

### 6.3 后续扩展

前端设计阶段可为每个点位增加个性化图片、封面图或视觉风格，但 V1 PRD 不强制要求图片素材。

## 7. 点位详情页

### 7.1 页面目标

点位详情页是 V1 的核心页面。用户进入某个点位后，应能从上到下快速获得明日日出概率与七日趋势。

### 7.2 信息展示顺序

点位详情页从上到下展示：

1. 山名，例如“梅里雪山”
2. 日期，例如“2026 年 6 月 4 日 星期四”
3. 更新时间，例如“更新时间：06-03 12:00”
4. 明日日出概率
5. 概率百分比，例如“概率 68%”
6. 日出时间 / 日落时间
7. 七日预报
8. 点击七日预报中的某一天，展开当天 EC、GFS、ICON 三种气象模型数据

### 7.3 明日日出概率

V1 主概率只针对“明日日出时段”的日照金山概率。

日落时间在页面中展示为参考信息，但 V1 不单独计算“日落金山概率”。

### 7.4 七日预报

七日预报默认展示：

- 日期
- 概率
- 趋势标签

七日预报用于辅助用户做行程判断。其中：

- 未来 1-3 天：作为重点预测。
- 未来 4-7 天：作为趋势参考。

### 7.5 三模型数据展开

用户点击七日预报中的某一天后，页面展开当天 EC、GFS、ICON 三种气象模型数据。

具体展示哪些字段暂不在 V1 PRD 中锁定，后续需要单独讨论。该部分先作为功能入口与信息结构保留。

## 8. 预测与刷新规则

### 8.1 刷新时间

V1 采用每天中午 12:00 统一刷新预测结果的规则。

刷新后，点位详情页展示各点位“明日日出概率”。

### 8.2 更新时间展示

页面需要展示最近一次预测结果的更新时间，格式示例：

`更新时间：06-03 12:00`

### 8.3 七日趋势

七日趋势随每天 12:00 的统一刷新一起更新。V1 不要求前端实时刷新多次模型数据。

## 9. 结果表达

V1 使用“概率百分比 + 推荐标签”的表达方式。

示例：

- `概率 78% 推荐`
- `概率 52% 谨慎`
- `概率 23% 不推荐`

推荐标签阈值暂不锁定，后续与“日照金山概率评分模型 V1”一起设计和校准。

## 10. 数据来源

V1 PRD 仅定义目标数据源，不锁定具体供应商、接口字段、调用频率或权重。

目标数据源包括：

- EC 气象模型
- GFS 气象模型
- ICON 气象模型
- 腾讯位置服务
- DeepSeek API

### 10.1 EC/GFS/ICON

用于提供各点位未来天气预测相关数据。三种模型的具体字段、数据接口、更新频率和融合方式后续专项讨论。

### 10.2 腾讯位置服务

V1 主要用于小程序地图定位、点位展示或导航能力。V1 不做地图任意选点预测。

### 10.3 DeepSeek API

DeepSeek API 计划用于气象数据分析、解释生成或用户可读建议输出。其具体职责暂不锁定，后续与概率评分模型一起讨论。

V1 不建议让 DeepSeek 单独决定最终概率。最终概率应保留可追踪、可复盘的计算依据。

## 11. 异常处理

### 11.1 最近成功结果优先

如果当天 12:00 刷新失败，但系统存在最近一次成功结果，则页面优先显示最近一次成功结果，并明确展示对应更新时间。

### 11.2 异常提示

当部分模型数据缺失或刷新异常时，页面应展示提示，例如：

`部分数据暂未更新，当前结果仅供参考。`

### 11.3 严重缺失

如果关键数据严重缺失，导致概率结果不可用，则隐藏概率结果，展示：

`数据更新中，请稍后查看。`

## 12. 设置页

### 12.1 页面目标

设置页用于承载产品说明、数据说明和用户反馈，降低用户误解与产品合规风险。

### 12.2 V1 内容

设置页包含：

- 关于产品
- 数据来源说明
- 免责声明
- 反馈入口

### 12.3 免责声明方向

免责声明需要说明：

- 概率结果仅供旅行参考，不构成气象预报承诺。
- 山区天气变化快，实际观测结果可能与预测不一致。
- 用户应结合当地天气、交通、安全提示自行判断出行安排。

## 13. V1 不做内容

以下内容不进入 V1：

- 地图任意选点预测
- 独立地图页
- 订阅提醒/推送通知
- 收藏点位
- 默认点位设置
- 旅行攻略内容
- 交通、住宿、机位推荐
- 详细概率算法设计
- EC/GFS/ICON 展开字段粒度设计
- 日落金山概率
- 专业气象图表

## 14. 待讨论事项

以下内容需要后续单独讨论或形成专项设计：

1. 日照金山概率评分模型 V1
2. EC/GFS/ICON 三模型数据的字段粒度
3. 三模型数据融合方式与权重
4. DeepSeek API 的具体职责边界
5. 推荐/谨慎/不推荐的阈值规则
6. 每个标杆点位的观测点、目标山峰、最佳观测方向与本地经验参数
7. 数据供应商、接口成本、调用频率与稳定性
- `供应商暂定Open-Meteo`
- `接口暂时不给钱，后续达到200收益后续费`
- `调用频率为一天四次`
- `使用外网vps，稳定接入外网api`
8. 后续是否增加地图任意选点预测`否`
9. 后续是否增加订阅提醒`否`

## 15. 成功指标

V1 可用性的核心判断标准：

- 用户进入小程序后，可以在较短时间内找到目标点位。
- 用户进入详情页后，可以快速理解明日日出概率与推荐程度。
- 用户可以查看未来七日趋势，并区分短期预测与趋势参考。
- 当数据异常时，页面不会误导用户，也不会无解释地空白。
- 产品边界清晰，不让用户误以为结果是官方气象预报或确定性承诺。

## 16. 版本结论

V1 先做一个轻量、清晰、面向游客的日照金山概率查询工具。产品重点不是一次性解决所有气象算法问题，而是先完成“点位选择 - 明日日出概率 - 七日趋势 - 推荐标签 - 数据说明”的最小闭环。

算法、模型字段、DeepSeek 分析边界和地图扩展能力后续再作为专项逐步完善。
